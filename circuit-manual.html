1 Introduction

Module digital-circuits.rkt provides a simple tool for the digital simulation of digital electronic circuits. A simulation is done by a procedure that given the digital inputs calculates the digital outputs. The procedure can have an internal state of digital values that are preserved between successive calls, which can be regarded as successive points of time at which the signals of the inputs are changed. The outputs and new internal state can depend on both the inputs and the preserved internal state. Ternary logic is used such as to include indeterminate signals:

Let n be the number of all wires in a circuit. The circuit cannot assume more than 3n distinct ternary digital states, usually much less. F and T are no booleans in the sense of predicate boolean?, id est (boolean? F) → #f and (boolean? T) → #f. Use predicates bit? and trit? in stead.

2 Introductory example

As introductory example a D-latch is shown.
It has two inputs, which we shall call ‘in’ and ‘clock’.
It has two outputs, which we shall call ‘state’ and ‘state-inverse’.
The state and state-inverse are preserved as internal state.
They always are inverses of each other.
After changing the inputs the new state can depend on the previous state.
The following transition table applies:

Hence, in order to set or reset the D-latch, set in to 1 cq 0 and apply a 1-pulse to the clock. Leave the clock low at 0 in order to preserve the state. There are several ways to construct a D-latch. The following diagram shows a D-latch consisting of four Nand-gates.

With syntax make-circuit-maker a procedure for simulation of the circuit can be made by straightforwardly describing the diagram:
> (define make-D-latch
   (make-circuit-maker
    'D-latch              ; name
    (in clock)            ; inputs
    (state state-inverse) ; outputs
    ; Gates:
    (reset         (Nand in    clock))
    (set           (Nand reset clock))
    (state         (Nand reset state-inverse))
    (state-inverse (Nand set   state))))
In fact syntax make-circuit-maker produces a thunk, called a circuit maker, that returns a procedure for the simulation, called a circuit procedure. We must call the circuit maker in order to make an instance of the circuit procedure proper. In section Make-circuit-maker we shall see why the circuit procedure is not returned immediately.
> (define D-latch (make-D-latch))
Procedure D-latch accepts the two arguments in and clock and returns the two values state and state-inverse. Via the wires state and state-inverse the state is fed back to itself via the two Nand-gates at the right in the diagram. When the clock is 0, both set and reset raise to 1 (as far as not already 1) and the two Nand-gates act as a stable loop of two inverters for the signals state and state-inverse. Once stable, the state and state-inverse are inverses of each other and remain as they are after the clock falls down from 1 to 0. Let’s test the D-latch for all combinations of binary values for in, clock and old state:

> (begin
   (define (action in clock)
    (cond
     ((= clock 0) 'preserve)
     ((= in    0) 'reset)
     ((= in    1) 'set)))
   ; 
   ; We print some details in a table.
   ; First the header of the table.
   ; 
   (printf " ~n")
   (define line "   ─────────────────────────────────────~n")
   (printf line)
   (printf "   in clock old-state new-state action~n")
   (printf line)
   ; 
   ; The test proper.
   ; 
   (for* ((clock bit-seq) (in bit-seq) (state bit-seq))
    ; First put the D-latch in old state state and check
    ; that the D-latch indeed assumes this state.
    (define-values (old-state old-state-inverse) (D-latch state 1))
    (unless
     (and
      (eq? old-state state)
      (eq? old-state-inverse (Not state)))
     (error "test fails"))
    ; Apply the D-latch to the combination of in and clock. Check it.
    (define-values (new-state new-state-inverse) (D-latch in clock))
    (unless
     (and

3 Make-circuit-maker

syntax

(make-circuit-maker name-expr (input ...) (output ...) gate ...)

 
input = id
output = id
gate = (gate-output gate-expr)
  | ((gate-output ...) gate-expr)
gate-output = id
 
  name-expr : (or/c symbol? #f)
  gate-expr : (values trit? ...)
Returns a thunk. Each time the thunk is called it returns a distinct instance of a circuit procedure, id est, an instance with its own internal state. This can be important when the circuit has preserved internal state and several instances of the circuit are needed in one or more other circuits; each such instance must have its own internal state in order that the instances do not perturb each others internal states.

The following restrictions apply:

• The inputs must be distinct identifiers.
• The outputs must be distinct identifiers.
• All gate-outputs must be distinct identifiers.
• The set of the inputs and that of the gate-outputs must be disjunct.
• The outputs must be a subset of the conjunction of inputs and gate-outputs.
• Within each gate-expr all identifiers of the inputs and gate-outputs are bound,
  those of other gates included. They can be used as inputs of the gate.
• Each gate-expr must return as many values as its gate has gate-outputs.
  These values will be the new ones for the gate-outputs.
• Assignments to inputs, outputs and gate-outputs are inhibited.
• A gate-expr must not involve direct nor indirect calls to the circuit it is part of.

The last three requirements are checked while a circuit procecure is running. The other requirements are checked immediately by syntax make-circuit-maker. The gate-exprs are not yet evaluated. They will be incorporated in the circuit procedures made by the thunk. The name-expr is evaluated immediately. The value is used for the object-name and the printed form of the circuit maker and the circuit procedures it makes. Here is an empty circuit:
> (define make-empty-circuit
   (make-circuit-maker 'nothing () ()))
> (define empty-circuit (make-empty-circuit))
> (values
   (list (object-name make-empty-circuit) make-empty-circuit)
   (list (object-name      empty-circuit)      empty-circuit))

(make:nothing #<circuit-maker:nothing>)

(nothing #<circuit:nothing>)

A gate-expr can include calls to gates and other circuits, but it must not directly nor indirectly call the circuit procedure it belongs to, for this would represent an infinitely deep nesting of the circuit, which in real life is impossible, of course. A circuit can have circular dependencies between its internal signals, though. For example the signals state and state-inverse in the above D-latch have such dependency. When a gate-expr directly or indirectly calls the circuit procedure it is part of, an exception is raised:
> (define make-nested-circuit
   (make-circuit-maker 'nested-circuit
    () () (() (nested-circuit))))
> (define nested-circuit (make-nested-circuit))
> (nested-circuit)

nested-circuit: direct or indirect recursive call

A gate-expr cannot make assignments to inputs or gate-outputs:
> (((make-circuit-maker 'wrong-circuit
     (var) () (() (set! var ?)))) ?)

wrong-circuit: assignment to input 'var' prohibited

> (((make-circuit-maker 'wrong-circuit
     () () (var (set!-values (var) ?)))))

wrong-circuit: assignment to gate-output 'var' prohibited

A gate is not required to have outputs. Such a gate can be used for inclusion of checks and for printed output as a probe on some or all wires:
> (define make-probed-circuit
   (let ((n 0))
    (make-circuit-maker 'probed-circuit () ()
     (a 1)
     (b (Not a))
     (c (Not b))
     (() ; probe:
      (begin
       (printf "Step ~s: a=~s, b=~s, c=~s.~n" n a b c)
       (set! n (add1 n))
       ; An assignment, but n is not an input nor a gate-output.
       ; set! returns #<void>, but we must not return any value.
       ; Therefore we finish with:
       (values))))))
> (define probed-circuit (make-probed-circuit))
> (probed-circuit)

Step 0: a=?, b=?, c=?.

Step 1: a=1, b=?, c=?.

Step 2: a=1, b=0, c=?.

Step 3: a=1, b=0, c=1.

The probe shows how the signal travels through the circuit from a via b to c.

4 Circuit procedure

Below circuit is supposed to be a circuit procedure as returned by a circuit maker made by syntax make-circuit-maker.

procedure

(circuit [#:report report    
  #:unstable-error unstable-error]    
  input ...) → 
trit? ...
  report : any/c = #f
  unstable-error : any/c = #t
  input : trit?
The inputs correspond to those given to syntax make-circuit-maker. Exacly as many inputs must be supplied as were given to make-circuit-maker. The circuit procedure returns a multiple value corresponding to the outputs given to make-circuit-maker. The combination of all gate-outputs is preserved as the internal state of the circuit. Initially, id est before the circuit procedure is called for the first time, all gate-outputs are ?, id est indeterminate. The circuit procedure makes steps as follows: all gate-exprs are evaluated. Each gate-expr must yield as many values as its gate has gate-outputs. The values must be trits of course. After all gate-exprs have been evaluated, the returned values are assigned to the gate-outputs without bothering about the delays that gates and subcircuits have in real life. More steps are made until all gate-outputs are stable. A gate-output is stable if it no longer changes value. After stability is obtained, the outputs are returned as a multiple value. The internal state is preserved for the next call to the circuit procedure.

When repetition of a previous unstable internal state is detected, the simulation is terminated, for otherwise the circuit procedure would loop forever. Because a circuit procedure has a finite number of feasible internal states and memorizes and inspects the history, loops always are detected. A circuit procedure always starts with empty history and clears its history before returning. This prevents a circuit procedure form halting prematurely when called repeatedly.

The outputs of a circuit can depend on the internal state. This is the case in the above D-latch. For such a circuit distinct instances of the circuit procedure are required wherever part of another circuit. This is the reason why syntax make-circuit-maker returns a thunk for the preparation of the circuit procedure proper. Each call to the thunk returns a distinct instance. An elementary gate or circuit is one whose outputs are fully determined by its inputs only, id est without dependency on a preserved internal state within the gate or circuit. The same elementary gate or circuit can be used multiple times in the same circuit or in two or more distinct circuits as though each occurrence refers to a distinct gate or circuit. Some caution is required, because make-circuit-maker cannot distinguish elementary gates or circuits from those with internal state.

If the circuit procedure is called with a true value for optional keyword argument report a report is printed on the current-output-port, showing mutations of the internal state after each step. It is allowed to ask for a report of a circuit procedure called by a gate-expr within a circuit procedure for which a report is asked for as well. This is discouraged, though, because it may result in a confusing nest of reports.

When instability is detected and unstable-error is not #f an exception is raised. If unstable-error is #f and instability is detected, the simulation is terminated, the possibly partially or fully undetermined outputs are returned and the possibly partially or fully undetermined internal state is preserved. In the following example instability is detected and an exception is raised:
> (define unstable-circuit
   ((make-circuit-maker 'unstable-circuit
     (a) (b) (b (And a (Not b))))))
; First initialize b:
> (unstable-circuit 0)

0

; The following report shows the loop when a = 1:
> (unstable-circuit 1 #:report #t #:unstable-error 'yes)

Report of circuit unstable-circuit:

Inputs:

a = 1

Initial state:

b = 0

Step 1 of unstable-circuit:

b : 0 -> 1

Step 2 of unstable-circuit:

b : 1 -> 0

unstable-circuit: repeated unstable state

  a = 1

  b = 0

syntax

(truth-table (id ...) expr maybe-omit-?)

 
maybe-omit-? = 
  | #:omit-?
The ids are bound within the expr. The list of ids must not contain duplicates. Keyword #:omit-? may appear before, among or after the other fields of the form, but must not appear more than once. Let n be the number of ids. If keyword #:omit-? is absent the expr is evaluated 3n times, one time for each combination of n trits. If keyword #:omit-? is present the expr is evaluated 2n times, one time for each combination of n bits. The result is a list of 3n cq 2n elements. Each element of this list reads:

(id ... (value ...))

where (value ...) are the values produced by the expr. For example:

 (1 0 (0 0))

 (1 1 (1 0))

 (1 ? (? 0))

 (? 0 (0 0))

 (? 1 (? ?))

 (? ? (? ?)))

9.1 Adder

We present a two’s complement 6-bit adder with overflow detection. There are 13 inputs: one carry-in, followed by the 6 bits of the first operand and finally the 6 bits of the second operand. The bits of the operands must be given in order of decreasing significance. There are 8 outputs: an overflow indicator, a carry-out bit and the 6 bits of the sum in decreasing order of significance. The overflow bit is 1 in case of overflow, else it is 0. If the carry-in is 0 the sum is calculated. If the carry-in is 1 the sum plus 1 is calculated.

> (define make-master-slave-flip-flop
   (let ((master (make-JK-flip-flop)) (slave (make-JK-flip-flop)))
    (make-circuit-maker 'master-slave-flip-flop
     (J K clock) ; inputs
     (P Q)       ; outputs
     ; The two JK-flip-flops, one for the master and one for the slave.
     ((set reset) (master (Nand (Nand J (Not K)) (Nand J Q))
                          (Nand (Nand K (Not J)) (Nand K P))
                          clock))
     ((P Q) (slave set reset (Not clock))))))
> (define master-slave-flip-flop (make-master-slave-flip-flop))

In order to clock the master-slave flip-flop we need two calls, one with clock=1 to set or reset or flip the master or to leave it as it is and one call with clock=0 in order to copy the state of the master into the slave. clock=1 may change the master but leaves the slave unaffected. clock=0 leaves the master unaffected and copies the state of the master into the slave. When clock=0, J and K are irrelevant and we take J=K=?.
> (define (clock-the-master-slave-flip-flop J K)
   (master-slave-flip-flop J K 1)
   (master-slave-flip-flop ? ? 0))
Let’s test clocking the master-slave flip-flop for all combinations of old state (P Q)
and the inputs J and K:
> (for* ((P bit-seq) (J bit-seq) (K bit-seq))
   (define Q (Not P))
   ; Put the flip-flop in state (P Q) and check it's done right.
   (define-values (old-P old-Q) (clock-the-master-slave-flip-flop P Q))
   (unless (and (= old-P P) (= old-Q Q)) (error "test fails"))
   ; Clock the flip-flop with J and K.
   (define-values (new-P new-Q) (clock-the-master-slave-flip-flop J K))
   (printf "(J=~s, K=~s, P=~s : new P=~s~n" J K old-P new-P)
   ; Check:
   (case (list J K)
    (((0 0)) (equal? (list new-P new-Q) (list old-P old-Q)))
    (((1 0)) (equal? (list new-P new-Q) (list 1 0)))
    (((0 1)) (equal? (list new-P new-Q) (list 0 1)))
    (((1 1)) (equal? (list new-P new-Q) (list old-Q old-P)))
    (else (error "test fails"))))
